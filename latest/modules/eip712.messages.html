eip712.messages¶

Message classes for typed structured data hashing and signing in Ethereum.

class eip712.messages.EIP712Message¶

Container for EIP-712 messages with type information, domain separator parameters, and the message object.

property body¶

The hash of the EIP-712 message (body_data).

property body_data¶

The EIP-712 structured message to be used for serialization and hashing.

property domain¶

The EIP-712 domain fields (built using HEADER_FIELDS).

property domain_type¶

The EIP-712 domain structure to be used for serialization.

property header¶

The EIP-712 message header.

property signable_message¶

The current message as a SignableMessage named tuple instance.

property version¶

The current major version of the signing domain. Signatures from different versions are not compatible.

class eip712.messages.EIP712Type¶

Dataclass for EIP-712 structured data types (i.e. the contents of an EIP712Message).

property data¶

Recursively built dict of the underlying data, to be used for serialization.

field_type(field: str) → str¶

Looks up field via type annotations, returning the underlying ABI type (e.g. "uint256") or EIP712Type. Raises KeyError if the field doesn’t exist.

types() → dict¶

Recursively built dict (name of type -> list of subtypes) of the underlying fields’ types.

class eip712.messages.SignableMessage(version: bytes, header: bytes, body: bytes)¶

These are the components of an EIP-191 signable message. Other message formats can be encoded into this format for easy signing. This data structure doesn’t need to know about the original message format.

In typical usage, you should never need to create these by hand. Instead, use one of the available encode_* methods in this module, like:

  • encode_structured_data()

  • encode_intended_validator()

  • encode_structured_data()

body: bytes¶

Alias for field number 2

header: bytes¶

Alias for field number 1

version: bytes¶

Alias for field number 0
